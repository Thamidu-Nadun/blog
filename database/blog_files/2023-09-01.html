Python Supply Chain Attack: A Growing Threat to Developers


NoName_13


In recent years, the cybersecurity landscape has witnessed a surge in supply chain attacks targeting software developers. These attacks exploit the trust that developers place in open-source packages and repositories to deliver malicious code to unsuspecting users. One such attack that has garnered significant attention is the deployment of rogue Python packages in the Package Index (PyPI) repository. In this article, we will delve into the details of these attacks, particularly focusing on the VMConnect campaign and its potential connection to North Korean state-sponsored threat actors.

The VMConnect Campaign: Exploiting Trust and Impersonation

The VMConnect campaign, first disclosed by ReversingLabs and Sonatype, revolves around a collection of Python packages that masquerade as legitimate open-source tools. These packages are designed to download an unknown second-stage malware onto the victim's system. The campaign recently made headlines when three additional rogue packages, namely tablediter, request-plus, and requestspro, were discovered in the PyPI repository.

To lure unsuspecting developers into installing these packages, the threat actors behind the campaign employ typosquatting techniques. By impersonating popular Python tools like prettytable and requests, they create a sense of trust and credibility. Once installed, these packages execute nefarious code that communicates with remote servers to retrieve and execute a Base64-encoded payload. The exact nature of this payload remains unknown.

One notable aspect of the VMConnect campaign is the evasion tactic employed by the tablediter package. Unlike previous iterations of the campaign, tablediter no longer triggers the malicious code immediately upon installation. Instead, it waits until the compromised application imports the designated package and calls its functions. This behavior-based detection evasion technique raises the bar for defenders and underscores the sophistication of the attackers.

The Role of request-plus and requestspro: Information Collection and C2 Communication

In addition to tablediter, the VMConnect campaign also utilizes two other rogue packages: request-plus and requestspro. These packages serve different purposes within the attack chain. request-plus is designed to collect information about the infected machine and transmit it to a command-and-control (C2) server. On the other hand, requestspro facilitates communication with the C2 server by sending a token received from the server and receiving a double-encoded Python module and a download URL in return.

The use of a token-based approach in the VMConnect campaign bears similarities to a previous npm campaign associated with North Korean actors. Microsoft-owned GitHub attributed these attacks to a threat actor it calls Jade Sleet, also known as TraderTraitor or UNC4899. This overlap in tactics suggests a common strategy employed by the adversaries to selectively deliver second-stage malware based on certain filtering criteria.

PyPI Attacks and North Korean Connections

The potential connection between the VMConnect campaign and North Korea is further supported by infrastructure overlaps and similarities with previous attacks. The npm engineering campaign, linked to North Korean actors, shares the token-based approach used in the VMConnect campaign. This approach, unique to these campaigns, has not been observed in malware hosted on public package repositories by other threat actors.

Moreover, the JumpCloud hack in June 2023, which also involved North Korean state-sponsored actors, revealed infrastructure overlaps with the npm campaign. These connections suggest a coordinated effort by North Korean threat actors to exploit software supply chains for their malicious activities.

The Role of py_QRcode: A Familiar Pattern

In the investigation of the VMConnect campaign, ReversingLabs discovered a Python package named pyQRcode that exhibits similarities to the VMConnect package. Interestingly, pyQRcode was previously associated with another North Korean activity known as SnatchCrypto or DangerousPassword. This Python malware, capable of running on Windows, macOS, and Linux, targets developers of cryptocurrency exchange businesses.

The discovery of py_QRcode and its connection to both the VMConnect campaign and SnatchCrypto suggests a pattern in the tactics employed by North Korean threat actors. They demonstrate a unique ability to write functional malware across multiple programming languages and target various operating systems. This adaptability enables them to maximize the impact of their attacks.

Protecting Against Python Supply Chain Attacks

As the VMConnect campaign and other Python supply chain attacks continue to emerge, it is essential for developers to remain vigilant and take proactive measures to protect their systems. Here are some recommendations to mitigate the risk of falling victim to such attacks:

  1. Verify Package Authenticity: Before installing any Python package, carefully review its source and ensure it comes from a trusted and reputable author. Verify the package's integrity by checking its digital signatures or checksums when available.

  2. Regularly Update Packages: Keep your Python packages up to date by regularly checking for new versions and security patches. Outdated packages may contain known vulnerabilities that threat actors can exploit.

  3. Implement Code Review Practices: Perform thorough code reviews to identify any suspicious or malicious code within the packages you use. Encourage collaboration and knowledge sharing among your development team to collectively identify potential risks.

  4. Monitor Package Dependencies: Maintain a comprehensive inventory of the packages your software relies on. Regularly monitor for any updates or changes in these dependencies to ensure they remain secure and trustworthy.

  5. Employ Security Solutions: Utilize security tools and solutions that can help detect and prevent supply chain attacks. These solutions may include static and dynamic code analysis, vulnerability scanning, and behavior-based detection.
